뮤텍스(Mutex)와 세마포어(Semaphore)

뮤텍스(Mutex)와 세마포어(Semaphore)는 둘 다 동기화(synchronization) 기법으로, 멀티스레드 환경에서 공유 자원에 대한 접근을 제어하기 위해 사용됩니다

1. 뮤텍스 (Mutex)
정의: 뮤텍스는 "Mutual Exclusion"의 약자로, 공유 자원에 대한 단일 스레드의 독점적 접근을 보장하는 동기화 객체입니다. 하나의 스레드가 뮤텍스를 잠그면, 다른 스레드는 그 자원을 사용할 수 없습니다.

특징:

소유권: 뮤텍스는 한 스레드만 잠글 수 있으며, 잠금이 해제되는 것도 그 스레드만 할 수 있습니다. 즉, 뮤텍스를 잠근 스레드만 그것을 해제할 수 있습니다.
상호 배제: 한 번에 한 스레드만 공유 자원에 접근할 수 있도록 합니다.
교착 상태(Deadlock)나 기아 상태(Starvation)가 발생할 수 있기 때문에 뮤텍스를 사용한 프로그램에서는 교착 상태를 방지하는 데 주의해야 합니다.
용도: 주로 데이터베이스, 파일 시스템, 특정 함수 호출 등에서 자원에 대한 동시 접근을 제어할 때 사용됩니다.


2. 세마포어 (Semaphore)
정의: 세마포어는 카운팅 방식으로 자원의 개수를 추적하면서 자원에 대한 접근을 제어하는 동기화 객체입니다. 세마포어의 값은 자원 수를 나타내며, 값이 0인 경우 추가적인 접근을 허용하지 않습니다.

특징:

카운팅 세마포어: 자원의 개수를 추적하며, 각 스레드는 자원을 사용할 때마다 세마포어 값을 감소시키고, 사용을 마친 후 값이 0보다 크면 다시 증가시킵니다.
바이너리 세마포어: 값이 0 또는 1인 세마포어로, 뮤텍스와 비슷한 동작을 하지만 세마포어는 여러 스레드가 동일한 리소스를 동시에 잠글 수 있기 때문에 소유권 개념은 없습니다.
동기화: 여러 스레드가 동시에 세마포어를 사용할 수 있기 때문에, 카운팅 세마포어는 주로 스레드 풀이나 제한된 수의 리소스를 제어할 때 사용됩니다.
용도: 주로 제한된 개수의 리소스를 관리하거나, 여러 프로세스 간의 동기화를 필요로 하는 경우 사용됩니다.


3.주요 차이점

항목	뮤텍스(Mutex)					세마포어(Semaphore)
소유권	소유권이 있음. 한 스레드만 잠그고 해제할 수 있음.	소유권이 없음. 여러 스레드가 값을 변경할 수 있음.
목적	공유 자원에 대한 단일 스레드 독점적 접근 보장		제한된 개수의 자원 관리 또는 동기화 기능 제공
카운팅	카운팅하지 않음 (단일 잠금)				카운팅 가능 (카운팅 세마포어)
용도	자원에 대한 단일 스레드 접근 제어			여러 스레드가 자원에 접근할 수 있도록 제한


4.결론: 
	뮤텍스는 하나의 스레드만 자원에 접근하도록 보장하는 반면, 
	세마포어는 여러 스레드가 자원을 공유하거나 자원의 개수를 제한하는 데 유용합니다.

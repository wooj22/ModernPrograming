POD(Plain Old Data)는 
C++에서 기존의 단순하고 전통적인 데이터 구조를 나타내는 유형입니다. 
POD는 C 스타일 구조체와 유사하며, C++의 강력한 객체지향 및 템플릿 특성을 사용하지 않는 단순 데이터 구조를 의미합니다.

C++11 이후부터는 POD 유형을 보다 정확히 정의하기 위해 몇 가지 기준이 추가되었습니다.

>>> POD의 조건

POD 유형은 다음 두 가지 조건을 모두 만족해야 합니다.

1) Trivial Type (중요한 작업이 없는 타입)

생성자, 소멸자, 복사 생성자, 복사 대입 연산자, 이동 생성자, 이동 대입 연산자를 명시적으로 정의하지 않아야 합니다.
기본적으로 제공된(컴파일러가 생성한) 생성자와 소멸자를 사용합니다.
가상 함수나 가상 상속을 포함하지 않아야 합니다.

2) Standard Layout Type (표준 레이아웃 타입)

모든 멤버가 동일한 접근 제어(public, private, protected)를 가져야 합니다.
멤버 변수는 기본 클래스와 파생 클래스 사이에서 순서가 뒤섞이지 않아야 합니다.
상속을 사용하더라도, 기본 클래스에 비공개 멤버가 없어야 합니다.
모든 멤버가 표준 레이아웃이어야 합니다.
이 두 조건을 모두 만족하면 해당 타입은 POD로 간주됩니다.


>>> POD의 특징

POD는 다음과 같은 특징을 가집니다:
메모리 레이아웃이 C 구조체와 호환됩니다.
초기화하지 않으면 데이터 멤버는 정의되지 않은 값을 가집니다.
멤버 오프셋이 일정합니다. 즉, C에서 offsetof를 사용할 수 있습니다.
C 표준 라이브러리 함수(memcpy, memcmp 등)로 안전하게 복사 및 비교할 수 있습니다.


>>> POD 유형의 용도

1) C와의 상호 운용성
POD는 C 스타일의 구조체와 동일한 메모리 레이아웃을 가지므로 C 함수와 데이터를 교환하는 데 사용됩니다.

2) 저수준 메모리 작업
memcpy와 같은 저수준 메모리 조작이 안전하게 가능합니다.

3) 최적화
POD는 간단한 메모리 레이아웃 덕분에 컴파일러가 더 효율적으로 최적화할 수 있습니다.


>>> C++11 이후의 변화

C++11에서는 POD의 정의가 더 명확히 구분되었고, 다음과 같은 유형이 추가되었습니다:

    Trivial Type: "단순하지만 표준 레이아웃을 가지지 않을 수도 있는 유형"을 의미.
    Standard Layout Type: "표준 레이아웃은 가지지만 단순하지 않을 수 있는 유형"을 의미.
    POD: 두 가지를 모두 만족하는 유형.


POD는 단순 데이터를 안전하고 효율적으로 관리하는 데 적합하며, 
C++11 이후에도 여전히 중요한 역할을 합니다. 
std::is_pod(C++17까지 사용 가능)과 같은 타입 특성을 사용하여 POD 여부를 확인할 수 있습니다. 
C++20부터는 대신 std::is_trivial과 std::is_standard_layout을 조합하여 사용합니다.
